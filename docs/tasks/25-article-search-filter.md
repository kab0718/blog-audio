# タグとキーワードで記事を検索できるようにする

## Task Summary

- Goal: 取得済み記事と URL 追加済み記事を、キーワードとタグでローカルに絞り込めるようにする
- User value: 記事一覧が増えても、聴きたい 1 記事 = 1 トラックをすばやく探せる
- Source issue: [#25 MVP: タグとキーワードで記事を検索できるようにする](https://github.com/kab0718/blog-audio/issues/25)

## Scope

### In scope

- 記事一覧画面の検索入力
- title / author / summary / tags / source label を対象にした即時フィルタ
- タグ選択による絞り込み
- キーワードとタグ条件の同時適用
- 検索条件のクリア
- 検索結果件数の表示
- 0 件時の検索専用空状態

### Out of scope

- provider API を使った全文検索
- 保存済み記事の高度な分類
- レコメンド

## Implemented Behavior

- 記事一覧画面に「記事を検索」入力とタグ絞り込みボタンを表示する。
- 検索は取得済みの `Article` 配列に対するクライアント側フィルタとして実行する。
- キーワードは正規化したうえで title / author / summary / tags / source label に照合する。
- タグボタンを選択すると、そのタグを持つ記事だけを表示する。
- キーワードとタグが同時に指定された場合は両方の条件を満たす記事だけを表示する。
- 検索後もキュー追加済み、現在再生中、音声生成状態は既存 view model と playback / track state から表示する。

## Acceptance Criteria

- キーワード入力で記事タイトル・著者・summary・タグに一致する記事だけが表示される。
- タグを選択すると、そのタグを持つ記事だけが表示される。
- 検索条件をクリアできる。
- 検索結果件数が画面上で分かる。
- 0 件時に通常の取得エラーと区別できる空状態が表示される。
- キュー追加済み、現在再生中、音声生成状態の表示が検索後も維持される。
- モバイル幅で入力欄、タグ、記事アクションが重ならない。
- `npm run build` が成功する。
